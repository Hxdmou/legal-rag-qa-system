# 贡献指南

欢迎您参与本项目的开发！我们非常欢迎社区的贡献，包括代码提交、文档改进、问题报告等。

## 📋 贡献流程

### 1. 提交 Issue
- 在提交代码之前，请先在 GitHub Issues 中搜索是否已有相关问题
- 如果没有，请创建新的 Issue 描述您的想法或问题
- Issue 标题清晰，描述详细

### 2. Fork 仓库
- 点击 GitHub 页面右上角的 "Fork" 按钮
- 将仓库克隆到本地：
```bash
git clone https://github.com/your-username/legal-rag-qa-system.git
cd legal-rag-qa-system
```

### 3. 创建分支
- 创建一个新的分支来开发您的功能或修复：
```bash
git checkout -b feature/your-feature-name
```

### 4. 开发与测试
- 编写代码，确保符合项目规范
- 运行测试确保功能正常
- 保持代码整洁，添加必要的注释

### 5. 提交代码
```bash
git add .
git commit -m "feat: 添加新功能描述"
git push origin feature/your-feature-name
```

### 6. 创建 Pull Request
- 在 GitHub 上创建 Pull Request
- 描述您的更改内容和目的
- 等待审核和合并

## 📝 代码规范

### Python 代码规范
- 遵循 PEP 8 编码规范
- 使用 4 个空格进行缩进
- 文件名使用小写字母和下划线
- 函数和变量名使用蛇形命名法（snake_case）
- 类名使用大驼峰命名法（CamelCase）
- 添加必要的类型提示

### 提交信息规范
```
类型: 简短描述（不超过50字符）

详细描述（如果需要）

相关 Issue: #123
```

**类型说明：**
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构（不新增功能也不修复 bug）
- `test`: 测试相关
- `chore`: 构建/工具相关

如果您发现了 bug，请按照以下格式提交 Issue：

```

**复现步骤:**

2. 步骤二
3. 预期结果
4. 实际结果

**环境信息:**
- 操作系统：
- Python 版本：
- 依赖版本：
```
