# 安全与隐私指南

本项目致力于保护用户数据安全和隐私。以下是关于数据存储、处理和安全的详细说明。

## 📁 数据存储位置

### 1. 对话历史数据
- **存储路径**: `./chat_histories/`
- **文件格式**: JSON
- **命名规则**: `{system_name}_history.json`
- **保留策略**: 默认保留最近 50 条对话记录

### 2. 知识库文档
- **存储路径**: `./knowledge_bases/`
- **支持格式**: PDF、DOCX、XLSX、TXT、MD 等

### 3. 向量索引数据
- **存储路径**: `./{system_name}_faiss_index/`
- **文件格式**: FAISS 索引文件

### 4. 日志文件
- **存储路径**: `./logs/`
- **文件格式**: 文本日志

## 🔒 安全操作指南

### 如何安全上传文档

1. **文件检查**:
   - 上传前检查文档内容，确保不包含敏感信息
   - 避免上传包含个人身份信息（PII）的文档

2. **文档脱敏**:
   - 对包含敏感数据的文档进行脱敏处理
   - 移除或替换姓名、身份证号、手机号、地址等信息

3. **访问控制**:
   - 确保只有授权人员可以访问服务器
   - 设置适当的文件权限

### 如何彻底删除数据

#### 删除对话历史
```bash
# 删除单个系统的对话历史
rm -rf ./chat_histories/{system_name}_history.json

# 删除所有对话历史
rm -rf ./chat_histories/
```

#### 删除知识库文档
```bash
rm -rf ./knowledge_bases/
```

#### 删除向量索引
```bash
rm -rf ./*_faiss_index/
```

### 数据加密建议

1. **传输加密**:
   - 使用 HTTPS 协议进行数据传输
   - 在生产环境中配置 SSL/TLS 证书

2. **存储加密**:
   - 对于包含敏感数据的部署，建议使用磁盘加密
   - 考虑对敏感配置文件进行加密存储

3. **配置加密**:
   - 使用环境变量管理敏感配置（如 API Key）
   - 不要将敏感信息硬编码到代码中

## 📋 安全最佳实践

### 1. 访问控制
- 限制对服务器和存储的访问权限
- 使用强密码和多因素认证

### 2. 定期清理
- 定期清理不再需要的对话历史和日志
- 建立数据保留策略

### 3. 日志管理
- 定期审查日志文件
- 设置日志轮换策略
- 避免在日志中记录敏感信息

### 4. 备份策略
- 定期备份重要数据
- 测试备份恢复流程
- 考虑异地备份

### 5. 更新维护
- 定期更新依赖库版本
- 关注安全漏洞公告
- 及时修复安全问题

## ⚠️ 安全风险提示

1. **数据泄露风险**:
   - 未经授权的访问可能导致数据泄露
   - 建议在生产环境中使用防火墙和访问控制

2. **模型安全**:
   - LLM 模型可能产生有害内容
   - 建议添加内容过滤和审核机制

3. **依赖安全**:
   - 第三方库可能存在安全漏洞
   - 建议定期检查依赖安全性

## 🔐 合规声明

本项目设计遵循以下合规原则：
- **数据最小化**: 仅收集必要的数据
- **数据主体权利**: 用户可以访问、更正、删除其数据
- **透明性**: 清晰说明数据处理方式
- **安全保障**: 采取适当的安全措施保护数据
