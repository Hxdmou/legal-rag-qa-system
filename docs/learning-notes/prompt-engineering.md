# 提示词工程学习笔记

## 一、提示词基础

### 1.1 什么是提示词工程

提示词工程是设计和优化提示词以获得更好模型输出的过程。

### 1.2 核心原则

- **清晰明确**：指令要具体、不含歧义
- **上下文相关**：提供必要的背景信息
- **格式规范**：使用结构化输出格式
- **角色设定**：为模型设定明确的角色

---

## 二、提示词设计模式

### 2.1 角色设定模式

```
你是一位专业的Python开发者，擅长代码优化和问题排查。

问题：请分析以下代码的性能问题并提供优化建议。

代码：
[代码片段]
```

### 2.2 上下文注入模式

```
基于以下文档内容回答问题：

[文档内容]

问题：[用户问题]

要求：
1. 仅基于提供的文档内容回答
2. 如果文档中没有相关信息，请明确说明
3. 回答要简洁明了
```

### 2.3 输出格式控制

```
请按照以下JSON格式输出结果：
{
    "answer": "回答内容",
    "confidence": 0.85,
    "sources": ["文档1", "文档2"]
}
```

## 三、RAG 专用提示词模板

### 3.1 基础模板

```
你是一个专业的问答助手。
请根据提供的上下文信息回答用户的问题。

上下文：
{context}

问题：{question}

要求：
1. 严格基于上下文信息回答
2. 不要编造信息
3. 如果无法回答，请说"无法从提供的信息中找到答案"
4. 回答要准确、简洁
```

### 3.2 进阶模板

```
你是一位专业的知识库问答专家。

## 任务说明
基于提供的参考文档，对用户问题进行准确回答。

## 参考文档
{context}

## 用户问题
{question}

## 回答要求
1. 必须引用参考文档中的内容作为回答依据
2. 对于不确定的信息，要明确标注
3. 如果文档内容冲突，优先选择最新的信息
4. 输出格式：
   - 答案：[你的回答]
   - 来源：[引用的文档名称]
```

## 四、优化技巧

### 4.1 分步骤思考

```
请按照以下步骤回答问题：

步骤1：理解问题的核心需求
步骤2：在上下文中查找相关信息
步骤3：综合分析找到的信息
步骤4：组织语言，给出清晰的回答

问题：{question}

上下文：{context}
```

### 4.2 示例引导

```
以下是回答问题的示例：

示例1：
问题：什么是RAG？
回答：RAG（Retrieval-Augmented Generation）是一种结合信息检索和生成模型的技术。

请按照类似的格式回答以下问题：

问题：{question}
```

## 五、常见问题

### Q1：如何让回答更简洁？

**解决方案**：
- 在提示词中明确要求"简洁回答"
- 使用分点列表格式
- 限制回答长度

### Q2：如何提高回答的准确性？

**解决方案**：
- 提供更丰富的上下文信息
- 使用结构化的提示词模板
- 添加事实核查要求
