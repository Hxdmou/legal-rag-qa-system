# RAG 检索增强生成技术学习笔记

## 一、RAG 技术概述

### 1.1 什么是 RAG

RAG（Retrieval-Augmented Generation）是一种结合信息检索与生成模型的AI技术。它通过以下流程工作：

1. **检索阶段**：从知识库中查找与用户问题相关的文档片段
2. **生成阶段**：基于检索到的信息，利用大语言模型生成准确的回答

### 1.2 RAG 的优势

- ✅ **减少幻觉**：回答基于真实文档，提高准确性
- ✅ **知识更新**：无需重新训练模型，更新知识库即可
- ✅ **可追溯性**：可以追踪回答的来源
- ✅ **领域适配**：适用于各种垂直领域

## 二、核心组件

### 2.1 文档处理流程

```
原始文档 → 文本分割 → 向量化 → 向量存储 → 检索匹配
```

### 2.2 关键技术

| 组件 | 作用 | 常用工具 |
|------|------|----------|
| 文本分割 | 将长文档切分为合适大小的chunk | LangChain RecursiveCharacterTextSplitter |
| 向量化 | 将文本转换为向量表示 | OpenAI Embeddings, DashScope Embeddings |
| 向量数据库 | 存储和检索向量 | FAISS, Pinecone, Milvus |
| 大语言模型 | 生成自然语言回答 | GPT, Qwen, Llama |

## 三、实践经验

### 3.1 Chunk 大小选择

- **小 chunk（200-500 tokens）**：检索精度高，但上下文可能不完整
- **大 chunk（1000-2000 tokens）**：上下文完整，但可能包含无关信息
- **建议**：根据文档类型和问答需求进行调整，通常 500-800 tokens 是不错的起点

### 3.2 检索策略

- **单向量检索**：简单直接，适合短问题
- **混合检索**：结合 BM25 和向量检索，提高召回率
- **查询扩展**：对用户问题进行多视角扩展，增加检索覆盖率

### 3.3 性能优化

1. **索引优化**：使用合适的向量索引类型
2. **缓存机制**：缓存高频查询结果
3. **异步处理**：文档处理和检索异步化

## 五、常见问题与解决方案

### 问题1：检索不到相关文档

**可能原因**：
- 文档切分不合理
- 向量化模型不匹配
- 查询表述不够准确

**解决方案**：
- 调整 chunk 大小和重叠比例
- 尝试不同的嵌入模型
- 使用查询扩展技术

### 问题2：回答不准确

**可能原因**：
- 检索结果质量差
- 提示词设计不合理
- 模型理解偏差

**解决方案**：
- 优化检索策略
- 改进提示词工程
- 添加上下文约束

### 问题3：性能瓶颈

**可能原因**：
- 向量数据库检索慢
- 文档处理耗时过长
- 模型调用延迟高

**解决方案**：
- 使用更高效的索引结构
- 预构建向量索引
- 考虑模型部署优化
