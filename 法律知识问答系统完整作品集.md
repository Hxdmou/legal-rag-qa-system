# 法律知识问答系统 - 完整作品集

## 项目概述

本项目基于检索增强生成（Retrieval-Augmented Generation, RAG）技术，实现了一个智能法律知识问答系统。系统支持上传多种格式的法律文档，自动建立向量索引，并基于文档内容进行精准的法律问题回答。

---

## 一、项目背景

在当今信息化时代，法律知识的获取和普及变得越来越重要。传统的法律咨询方式存在成本高、效率低、覆盖面有限等问题。为了解决这些痛点，本项目利用人工智能技术，开发了一个基于RAG技术的智能法律知识问答系统。

**项目目标：**
- 降低法律咨询门槛，让普通民众能够快速获取法律知识
- 提高法律文档的检索和利用效率
- 为法律从业者提供智能化的辅助工具

## 三、功能特性

### 3.1 文档管理功能

| 功能 | 说明 |
|------|------|
| 多格式支持 | 支持TXT、PDF、Excel（.xlsx, .xls）等常见文档格式 |
| 批量上传 | 支持多文件同时上传，提高效率 |
| 知识库管理 | 可视化展示当前知识库文件，支持单个或批量删除 |
| 动态更新 | 上传新文档后自动更新知识库索引 |

### 3.2 智能问答功能

| 功能 | 说明 |
|------|------|
| 语义理解 | 理解用户提出的法律问题 |
| 精准检索 | 从知识库中检索最相关的文档片段 |
| 智能回答 | 基于检索结果生成准确的法律回答 |
| 多轮对话 | 支持上下文连贯的多轮对话交流 |

### 3.3 对话历史管理

| 功能 | 说明 |
|------|------|
| 自动保存 | 对话内容自动保存到本地文件 |
| 历史查询 | 可查看和回溯历史对话记录 |
| 一键清空 | 支持清空对话历史，保护隐私 |

## 五、技术实现

### 5.1 核心技术栈

| 层次 | 技术选型 | 说明 |
|------|----------|------|
| 前端展示 | Streamlit | 快速构建交互式Web应用 |
| 文档解析 | PyPDF/Excel库 | 支持多种格式文档内容提取 |
| 文本处理 | NLP分块技术 | 智能文本分割和预处理 |
| 向量化 | Embeddings模型 | 将文本转换为向量表示 |
| 向量存储 | FAISS/向量数据库 | 高效的相似度检索 |
| 大语言模型 | 通义千问/OAI | 智能问答生成能力 |

### 5.2 关键技术流程

**文档处理流程：**
```
用户上传文档 -> 文档解析 -> 文本分块 -> 向量化 -> 存储到向量数据库
```

**问答处理流程：**
```
用户提问 -> 问题向量化 -> 相似度检索 -> 上下文组装 -> LLM生成回答 -> 返回结果
```

## 六、使用说明

### 6.1 环境准备

- Python 3.8+
- 必要的依赖库（见requirements.txt）
- 支持的操作系统：Windows、Linux、MacOS

### 6.2 启动步骤

**步骤1：安装依赖**
```bash
pip install -r requirements.txt
```

**步骤2：启动系统**
```bash
streamlit run medical_qa_fixed.py --server.port 7869
```

**步骤3：访问系统**
打开浏览器访问 http://localhost:7869

### 6.3 操作指南

**首次使用流程：**
1. 打开系统网页
2. 在左侧上传法律文档（PDF、TXT、Excel格式）
3. 等待系统处理完成（显示"知识库更新完成"）
4. 在底部输入框输入法律问题
5. 系统自动生成回答

**注意事项：**
- 首次使用前需先上传法律文档建立知识库
- 支持多种格式文档混合上传
- 对话历史会自动保存
